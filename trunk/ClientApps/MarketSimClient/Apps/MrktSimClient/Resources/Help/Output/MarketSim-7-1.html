Create/Edit a Simulation

Before a scenario simulation session can be conducted, it is necessary to set parameters for the scenario simulation. For this reason, the MarketSim scenario configuration tools are provided as the Create New Simulation... option of the Scenario menu.

Navigation: From MarketSim front page-->Database <name>--> Project: Models <click to select one> --> Scenarios <double-click to select one> = Scenario/Simulations/Results page -->Scenario, Create New Simulation = Editing <name of scenario> 3-tab window

Scenario Simulation Settings

From the Simulation tab, you can check the start and end dates of the scenario and how often data is aggregated. You can also change the random seed used to drive the simulation as well as specify that metrics be pre computed. In most cases you do not need to pre compute metrics, MarketSim will warn if you create a scenario without any metrics being pre computed (but this is not an error.)

Drop-down selection range from which to set one of the following simulation types:

  • Standard-Runs the simulation to form a single set of results.
  • Parallel Search-Uses variables to generate multiple results.
  • Serial Search-If multiple parameters are used, this option changes them one at a time (sequentially).
  • Random Search-Randomly chooses values for variables.
  • Statistical-Generates multiple results with difference.
  • Checkpoint-Creates a checkpoint for seeds model.
Write Data

Frequency by which data is to be aggregated, expressed as a multiple-ranging from 1 to 10-of daily, weekly, or monthly. The rate at which runs occur is based on the start/end dates set for the simulations.

Note: To enable faster MarketSim outputs and to prevent stress to your computer's hard drive, run data less frequently.

Simulation Scaling

Value to indicate the percentage of total consumer population to be simulated, as listed in the model summary.

Note: Higher percentages equal slower run times during simulation.

Acceptable Value(s): 0 - 100

Random Seed

A numeral by which to initialize the random seed generator.

Creating a Checkpoint Scenario

MarketSim provides the checkpoint scenario option to enable setup of a scenario representative of a checkpoint in time. Using a checkpoint scenario allows comparison from which to determine the effects, over time, of the various parameters used to run various simulations. You can create any number of checkpoint scenarios for a model, but currently only one checkpoint can be active at a time. The model summary displays the checkpoint time, if one has been created.

A checkpoint scenario can be created from any existing scenario. To do so, edit a selected scenario and use the Checkpoint option from the scenario type drop-down menu . All checkpoints currently created are displayed in the model summary.

Navigation: From MarketSim front page-->Database <name>--> Project: Models <click to select one> --> Scenarios <double-click to select one> = Scenario/Simulations/Results page -->Scenario, Create New Simulation = Editing <name of scenario> 3-tab window, simulation tab

The End Date specified for a checkpoint scenario becomes the checkpoint date. If a model has a checkpoint, any scenario that has a start date that either begins on or after the checkpoint date will use the checkpoint. If the checkpoint occurs prior to the EndDate, the simulation will run from the model Start Date.

Once you have created a checkpoint scenario, you can run it in the same way as you would run any scenario in MarketSim. A checkpoint that runs successfully to completion results in display of Checkpoint created in the simulation status reports.

Setting Simulation Parameters

From the Parameters tab, you can see the product plans are currently available for use with this scenario (in the uppermost view area). You can set any product in view to be either available [upper portion of window) or associated (lower portion of window)

  • To use a product plan in the simulation, click to highlight a product plan from the top pane and click the Add Parameter to Simulation button, which will move the selection to the lower pane of the window.
  • Conversely, to remove the association, click to highlight the product plan from the lower pane of the window and click Remove Parameter from Simulation to reposition the selection into the available product plans view area.

To delete the model: cancel out of editing the scenario and from the Model menu select "Delete Model"

Button that moves the highlighted row from the top section of the window to the bottom portion. When moved into the bottom, the selection will be included in the simulation.

Button that moves a highlighted row from the bottom portion of the window to the upper portion. When removed, from the bottom, the selection is no longer a candidate for the simulation.

Button to reset modified parameter value(s), as based on the setting for the model. The reset value is displayed at the lower portion of the Parameters window.

Setting Simulation Variables

Use the Variables tab to set controls to be applied to the scenario parameters during simulation. To set up a new variable, click the Create New Variable...button to present the Enter Mnemonic for Variable dialog box. After dismissing this dialog box (click the x in the upper right corner), your variable name is entered into the Variables list and includes default values for min, max, rum_steps, and type. Click in any column to modify variable values.

The window shown in this section is selectable either from Scenario, Create New Simulation... or from Simulations, Edit Simulation (for a highlighted scenario) : using the Variables tab

Controlling Simulations

Once you have completed settings for simulation of a scenario, you can activate a simulation run for it. As you continue to accumulate simulation runs, information about each is displayed at the Scenario/Simulation/Results window . Each of the simulations in the Simulations list can be controlled by options available from the Simulations menu, as described in the following topics:

  • Run Simulation...
  • Edit Simulation
  • Copy Simulation
  • Delete Simulation

Run Simulation...

A simulation run is necessary before charts and graphs can generated to represent modeled data.

Table 19 Run Simulation

Copy Simulation

Copying a simulation replicates it in the database, and automatically assigns it a name and increments the name of the simulation by a value of 1. Correspondingly, it becomes the next listed simulation in the Simulation list.
