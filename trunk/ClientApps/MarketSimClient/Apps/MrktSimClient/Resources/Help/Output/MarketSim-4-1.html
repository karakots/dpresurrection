Check System Requirements

Ensure that the computer that will be running MarketSim is loaded with the proper Windows operating system and has sufficient memory installed:

  • Windows XP, Service Pack 2
  • Intel Pentium 1.60GHz processor, minimum 512 MB RAM (recommended)
  • .NET 2.0 (available from Microsoft)

Get System Data Management Tools

MarketSim operations require the MS SQL Server database software platform.

Note: If your computer already contains the MS SQL Server, skip this section, and proceed to the section entitled "Install MarketSim" on page 3.

Note: To load the MS SQL Server on your computer, use the downloads provided at the SQL Server 2005 Express Edition page provided by Microsoft:

MS SQL Server installation guidelines are provided in the Microsoft documentation. If you encounter problems, contact Decision Power for additional assistance. Installation of MS SQL Server database software results in loading the MS SQL server tools listed in the following table.

Install MarketSim

Completion of the Decision Power software installation results in placement of the MarketSim 2.x application in your Windows Start menu. Initial launch of the MarketSim program presents the MarketSim front page .

The MarketSim front page enables access to the tools you need for completion of the MarketSim installation. Completion of the installation requires the you define both the client database and the simulation engine on which to run MarketSim configuration sessions, as described in the following topics:

  • Configure the Client Database (on page 4)
  • Set Up the Simulation Engine (on page 9).

Configure the Client Database

After you have installed MarketSim, the MarketSim interface will walk you through the necessary steps to set up a client database, as described in this section. The MarketSim Client application requires connectivity to a MarketSim database. You can set up multiple databases, but only one database can be used at a time. Use the steps provided in this section to create a new database for connection to the MarketSim client application.

Steps to set up this connection must be performed in the following sequence:

1.For a new database, use the Data Link Properties window to set the database provider name to be used by MarketSim. (on page 4)
2.Set Database name. (on page 5)
3.Apply and verify the new database name. (on page 6)

1. For a new database, use the Data Link Properties window to set the database provider name to be used by MarketSim.

Navigation:

Start -> All Programs->MarketSim2.x = MarketSim launch window. Click Connect to database...to present the MarketSim Connection Manager window.Click Create New Connection to present the Data Link Properties panel.

2. Set Database name.

The Connection Manager now displays the name of the server, which you entered in the previous step. If successfully connected to server, you can now bind one more databases to the server.

Navigation:

Right-click on database provider name to present the New ModelDb... popup menu.

Select the menu option to present the New ModelDb dialog box. At the New ModelDb dialog box, type the database name, and click Create Connection to database... Wait for MarketSim to create the new database. While you wait, the Connection Manager displays status messages. Wait until the Data Link Properties panel is presented, and proceed to "Apply and verify the new database name." on page 6.

3. Apply and verify the new database name.

Set Up the Simulation Engine

The simulation engine setup requires that you define the data source, as described in the following topics:

  • Create a Data Source for Your User Name on This PC
  • Bind the Data Source to the SQL Server

Create a Data Source for Your User Name on This PC

1. Access the Create Data Source setup tools

  1. Launch the Simulation Engine to present the Select Data Source window.
  2. Make sure the Machine Data Source tab is selected, and click New... to present the Create Data Source wizard .

2. Select Data Source type as User Data Source (Applies to this machine only).

This is the default setting for the data source. Click Next> to present the driver selection window.

3. At the driver selection window , highlight SQL Server, then click Next> to view the summary of your settings.

4. At the summary window, make sure that the correct data source and driver are displayed, then click Finish to complete the create data source operation.

Bind the Data Source to the SQL Server

Your confirmation of the data source configuration enables access to the SQL Server connectivity screens described in this section.

1. Set data source and computer associations for the SQL server.

At the first of the SQL server configuration screens , identify the connection name, and name of your computer. Optionally, you can type a description of this connection.

2. Apply Windows NT authentication and source settings for configurations from the SQL server.

A dialog box now queries for method of SQL server verification with the login ID.

3. Identify the default database.

Apply default database settings to identify the database name and the ANSI requirements.

4. Set character data translation.

Click to select Perform translation for character data .

5. Check your entries and run a connectivity test that uses your approved entries.

Use the ODBC database summary report dialog boxes to test the data source and confirm your entries.

6. Apply the machine data source. At the Select Source dialog box , click to highlight mrktsimdb, then click OK. The simulation should now run.

Importing a Model

Before you can use the MarketSim model it needs to imported into your database. The process for importing a model into MarketSim requires that you uncompress the model and validate the import, as described in this section. You can use the model to create various projects, including a checkpoint (see also "Creating a Checkpoint Scenario" on page 40).

Uncompress the Model

The MarketSim model provided by DecisionPower is initially compressed. You use the MarketSim import function to uncompress the file-turning it into a.msdb (MarketSim data base) file- so that MarketSim can read it.

1. Launch MarketSim and open the project you wish to import the model into.

From the Model menu, choose Import... to present the Open window .

2. At the Open window, choose the.msdb file of the uncompressed msdb file and click Open. Status of the import is now reported at the bottom of the project window. Wait for completion of the import activity.

Validating the import

All models loaded into MarketSim are listed in the Project Models pane of the MarketSim window. To validate an import model:

1. Click to highlight the name of a model, to view the list of associated scenarios.

2. Double-click a scenario name to present the Scenario/Simulations/Results window from which to put a scenario to work.
