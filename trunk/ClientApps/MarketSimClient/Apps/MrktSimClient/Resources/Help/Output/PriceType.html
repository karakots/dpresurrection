MarketSim Price Types                       Table of Contents

In MarketSim you can create prices with a variety of properties. During a simulation, when a consumer goes shopping, MarketSim determines which products are available in the store and what the prices are. Only one price can be assigned to a product at a time. In addition, each price can also influence the consumer with regard to the awareness of the product and adding to the overall persuasion for the product. Finally, a utility can be added to the product. This utility will only be in effect for the current purchase.

Price Type Parameters

Each price type is defined by the following parameters. These values can be set in the model editor under ProductsàPrice Types.

Relative

This flag determines if  the price is evaluated as referenced to the current un-promoted price. This is evaluated as a percentage. If the un-promoted price is 1.5 and the value was -50 then the price the consumer would see would be 1.

If the relative flag is set then

Actual Price = ( 1 + (relative price value)/100) x (current un-promoted price)

Awareness

At the time of purchase and if the consumer is not already aware of the product then the price awareness value may cause the consumer to be aware. This is identical to the manner in which all awareness values are processed. It should be noted that the price awareness is processed separately from the distribution awareness, one represents the awareness driven by the product being on the shelf, and the other is driven by any special pricing displays.

Persuasion

Similar to awareness, persuasion is also processed. Again this is identical to the how persuasion is processed for distribution or any other media.

BOGN

If the price is a “Buy One Get 1 Free” then the BOGN value should be 1. If it is “Buy one get 2 Free” then this value should be 2, and so on. A value of zero does not affect the number of units purchased.

Segment Utility

For each price type and for each segment an associated utility is assigned. By default this is zero, but this can be set to represent the added utility this displayed price lends to the product. The segment price utilities can be found in a tab in the segments panel.

Un-promoted Price

The un-promoted price is a special price type that does not allow any parameters to be set for it, the segment utility is by default set to zero.

NOTE:  If the un-promoted price is not set or is zero for a given product, then the product will be de-activated and not available for purchase.

Using Price Types in Market Plans

Every price component in a media plan references a price type.

It is important to note that despite the fact that an end date can be set for a price component row this does not set the price to zero or the price distribution to zero after this date.

All prices retain their current price values until a new price values is set. To turn off a price type in a given component set the distribution value to zero.
